# 제품 요구사항 정의서 (PRD): 전화번호 관리 시스템

## 1. 프로젝트 개요
사용자가 개인 연락처를 안전하고 효율적으로 저장, 관리, 검색할 수 있는 웹 기반 시스템입니다. Supabase를 백엔드로 사용하며 Next.js 프레임워크를 통해 빠르고 보안이 강화된 사용자 경험을 제공합니다. 특히 개인정보 보호를 위해 이름과 전화번호는 암호화하여 저장합니다.

## 2. 사용자 스토리
- **사용자**로서, 새로운 연락처(이름, 번호, 메모)를 안전하게 등록하고 싶다.
- **사용자**로서, 등록된 연락처 정보를 수정하거나 불필요한 연락처를 삭제하고 싶다.
- **사용자**로서, 암호화된 상태에서도 이름이나 전화번호의 일부 입력으로 연락처를 검색하고 싶다.
- **사용자**로서, 많은 연락처를 지연 없이 쾌적하게 조회하고 싶다.

## 3. 기능적 요구사항
### 3.1 연락처 관리
- **추가**: 이름(필수), 전화번호(필수), 카테고리(필수), 메모(선택)를 입력하여 저장. 저장 시 이름과 전화번호는 암호화 처리됨.
- **카테고리 분류**: 가족, 친구, 친척, 동아리, 직장동료 등 사용자가 선택할 수 있는 카테고리 제공.
- **수정**: 기존 연락처의 정보를 수정 가능. 수정 시에도 암호화 및 카테고리 정책 유지.
- **삭제**: 사용자 확인 후 연락처 완전 삭제.
- **조회**: 전체 사용자 목록 표시 (복호화되어 화면에 출력).

### 3.2 검색 및 필터링 기능
- 이름 또는 전화번호의 부분 일치 검색 기능 제공.
- 카테고리별 필터링 기능 제공 (예: '가족' 카테고리만 보기).
- 암호화된 데이터를 검색하기 위한 효율적인 인덱싱 또는 서치 토큰(Search Token) 기법 고려.

### 3.3 목록 표시
- 무한 스크롤(Infinite Scroll) 방식을 사용하여 데이터 로딩 최적화.

## 4. 기술 요구사항 (기술 스택)
- **프론트엔드**: Next.js (App Router), React, Tailwind CSS.
- **백엔드/DB**: Supabase (PostgreSQL).
- **배포**: Vercel.
- **데이터 보안**: AES-256 등 표준 암호화 알고리즘 적용 (Application-level 또는 DB-level 전용 확장 도구 사용).

## 5. 보안 및 비기능 요구사항
- **데이터 암호화**: 개인정보(이름, 전화번호)는 DB 저장 시 반드시 암호화되어야 하며, 권한이 있는 사용자만 복호화하여 볼 수 있어야 함.
- **SQL 주입 방지**: Supabase Client SDK 활용 및 Row Level Security(RLS) 적용.
- **데이터 유효성 검사**: 프론트엔드 및 백엔드 양측에서 검증 수행.
- **환경 변수 관리**: 암호화 키 등 민감 정보는 Vercel/Supabase 설정에서 엄격히 관리.

## 6. 데이터베이스 스키마 (초안)
| 컬럼명 | 타입 | 제약 조건 | 설명 |
| :--- | :--- | :--- | :--- |
| id | UUID | 기본키 (PK) | 고유 식별자 |
| name | Text | Not Null | 연락처 성함 (암호화 저장) |
| phone | Text | Not Null | 전화번호 (암호화 저장) |
| category | Text | Not Null | 카테고리 (가족, 친구 등) |
| memo | Text | | 추가 메모 |
| created_at | Timestamptz | 기본값: now() | 생성 일시 |

## 7. UI/UX 디자인 요구사항
- **디자인 컨셉**: Facebook 스타일의 청정하고 신뢰감 있는 UI.
- **색상 팔레트**: Facebook Blue (#1877F2)를 포인트 컬러로 사용하고, 배경은 밝은 그레이 또는 화이트 톤 유지.
- **레이아웃**: 상단 네비게이션 바(검색바 포함), 카드 형태의 연락처 목록, 사이드바를 통한 카테고리 필터링.
- **컴포넌트**: 둥근 모서리(8px-12px), 그림자 효과를 최소화한 카드 디자인, 깔끔한 폰트(Inter 또는 시스템 폰트).
- **반응형**: 모바일 기기에서도 Facebook 앱과 유사한 쾌적한 사용성 제공.
